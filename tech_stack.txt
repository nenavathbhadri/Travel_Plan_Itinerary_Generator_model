1. BACKEND TECHNOLOGIES
=====================================================================

1.1 Python 3.13
-----------------
What    : Python is a high-level, general-purpose programming language.
How Used: It is the core programming language of this entire project.
          All backend logic, database operations, API calls, AI model
          inference, and server-side processing are written in Python.
Where   : Every backend file — views.py, urls.py, settings.py, __init__.py


1.2 Django 6.0 (Web Framework)
-------------------------------
What    : Django is a high-level Python web framework that follows the
          MVT (Model-View-Template) architectural pattern. It handles
          URL routing, request/response processing, template rendering,
          session management, and security (CSRF protection).
How Used: Django serves as the backbone of the web application.
          - URL Routing: Maps URLs to view functions (urls.py)
          - Views: Handles business logic for each page (views.py)
          - Templates: Renders HTML pages with dynamic data
          - Sessions: Manages user login state (request.session)
          - CSRF: Protects forms from cross-site request forgery
Where   : 
          - Itinerary/settings.py    → Project configuration
          - Itinerary/urls.py        → Root URL configuration
          - ItineraryApp/urls.py     → App-level URL routing
          - ItineraryApp/views.py    → All view functions
          - ItineraryApp/templates/  → All HTML templates


1.3 MySQL (Database)
---------------------
What    : MySQL is an open-source relational database management system.
          It stores data in structured tables with rows and columns.
How Used: MySQL replaced SQLite as the primary database. It stores:
          - User registration data (username, hashed password, email, etc.)
          - Trip history (source, destination, budget, itinerary text)
          - Admin credentials (username, hashed password)
          All database operations use SQL queries via PyMySQL.
Where   :
          - Database name: "itinerary"
          - Table "register"     → User accounts
          - Table "trip_history"  → Saved itineraries per user
          - Table "admin_users"   → Admin login credentials
          - Configured in: .env file (MYSQL_HOST, MYSQL_USER, etc.)


1.4 PyMySQL (MySQL Connector)
------------------------------
What    : PyMySQL is a pure-Python MySQL client library. It allows
          Python code to connect to and interact with MySQL databases.
How Used: PyMySQL establishes connections to the MySQL server and
          executes SQL queries (SELECT, INSERT, DELETE, CREATE TABLE).
          It uses DictCursor so query results are returned as
          dictionaries for easy access (e.g., row['username']).
Where   :
          - Itinerary/__init__.py  → Registers PyMySQL as MySQLdb
          - ItineraryApp/views.py  → get_db_connection() function
            Used in: UserLoginAction, RegisterAction, TravelPlanAction,
            TripHistory, AdminLoginAction, AdminDashboard, AdminDeleteUser


1.5 Groq API + LLaMA 3.3 70B (AI Engine)
------------------------------------------
What    : Groq is a cloud AI inference platform. LLaMA 3.3 70B is a
          large language model (LLM) by Meta with 70 billion parameters.
          Groq provides ultra-fast inference for this model via API.
How Used: When a user submits trip details (source, destination, budget,
          preferences), the app sends a structured prompt to the Groq API.
          The LLM generates a detailed day-by-day travel itinerary
          including activities, costs, food recommendations, and tips.
Where   :
          - ItineraryApp/views.py → TravelPlanAction() function
          - API key stored in: .env file (GROQ_API_KEY)
          - Model used: "llama-3.3-70b-versatile"


1.6 Google Gemini API (Fallback AI)
------------------------------------
What    : Google Gemini is Google's generative AI model for text,
          image, and multimodal content generation.
How Used: Acts as a fallback content generation engine. If the Groq
          API fails or is unavailable, Gemini can be used to generate
          itineraries as an alternative.
Where   :
          - ItineraryApp/views.py → Configured as backup
          - API key stored in: .env file (GOOGLE_API_KEY)


1.7 RAG Model (Retrieval-Augmented Generation)
------------------------------------------------
What    : RAG combines a retrieval system with a generative model.
          It first retrieves relevant documents/data, then uses that
          context to generate more accurate and grounded responses.
How Used: The RAG model uses HuggingFace Transformers to load a
          pre-trained model that matches user trip queries against
          existing travel plan templates. This enhances the quality
          of generated itineraries by grounding them in real data.
Where   :
          - ItineraryApp/views.py → Model loaded lazily on first use
          - Uses: transformers library (AutoTokenizer, AutoModel)


1.8 PyTorch (Machine Learning Framework)
------------------------------------------
What    : PyTorch is an open-source deep learning framework developed
          by Meta. It provides tensor computation and neural network
          support for training and running ML models.
How Used: PyTorch is the backend engine for the RAG model. It handles
          the tensor operations, model inference, and embeddings
          computation when the RAG model processes trip queries.
Where   :
          - Used internally by the transformers library
          - Loaded when RAG model initializes in views.py


1.9 NLTK (Natural Language Toolkit)
-------------------------------------
What    : NLTK is a Python library for natural language processing.
          It provides tools for tokenization, stemming, stopword
          removal, and text preprocessing.
How Used: NLTK preprocesses user input text (trip descriptions) by
          removing stopwords (common words like "the", "is", "and")
          and cleaning the text before sending it to the AI models.
          This improves the quality of generated itineraries.
Where   :
          - ItineraryApp/views.py → Text preprocessing functions
          - Data: nltk.corpus.stopwords (English stopwords list)


1.10 hashlib (Password Hashing)
---------------------------------
What    : hashlib is a built-in Python module that provides secure
          hash functions including SHA-256, SHA-512, MD5, etc.
How Used: The SHA-256 algorithm hashes user passwords before storing
          them in the database. During login, the entered password
          is hashed and compared with the stored hash. This means
          plain-text passwords are NEVER stored in the database.
Where   :
          - ItineraryApp/views.py → hash_password() function
          - Used in: RegisterAction (hashes before INSERT)
          - Used in: UserLoginAction (hashes before comparison)
          - Used in: AdminLoginAction (hashes before comparison)


1.11 python-dotenv (Environment Variables)
-------------------------------------------
What    : python-dotenv reads key-value pairs from a .env file and
          sets them as environment variables, keeping sensitive data
          (API keys, database passwords) out of source code.
How Used: Loads API keys and database credentials from the .env file
          so they are not hardcoded in the Python source files.
Where   :
          - .env file contains: GROQ_API_KEY, GOOGLE_API_KEY,
            MYSQL_HOST, MYSQL_USER, MYSQL_PASSWORD, MYSQL_DATABASE
          - Loaded in: ItineraryApp/views.py → dotenv.load_dotenv()

4. DATABASE SCHEMA
=====================================================================

Database: itinerary (MySQL)

Table 1: register
------------------
Column      | Type         | Description
------------|--------------|----------------------------------
username    | VARCHAR(50)  | Primary key, unique username
password    | VARCHAR(128) | SHA-256 hashed password (64 chars)
contact_no  | VARCHAR(20)  | User's phone number
email       | VARCHAR(50)  | User's email address
address     | VARCHAR(65)  | User's city or address

Table 2: trip_history
----------------------
Column      | Type         | Description
------------|--------------|----------------------------------
id          | INT (AUTO)   | Primary key, auto-increment
username    | VARCHAR(50)  | Foreign key → register.username
source      | VARCHAR(100) | Trip starting location
destination | VARCHAR(100) | Trip destination
budget      | VARCHAR(20)  | Trip budget amount
description | TEXT         | User's trip preferences/notes
itinerary   | LONGTEXT     | Full AI-generated itinerary text
created_at  | TIMESTAMP    | When the trip was generated

Table 3: admin_users
----------------------
Column      | Type         | Description
------------|--------------|----------------------------------
id          | INT (AUTO)   | Primary key, auto-increment
username    | VARCHAR(50)  | Unique admin username
password    | VARCHAR(128) | SHA-256 hashed password

5. SECURITY MEASURES
=====================================================================

5.1 SHA-256 Password Hashing
    - All passwords (user + admin) are hashed using SHA-256 before storage
    - Plain-text passwords are NEVER stored in the database
    - Login compares hash of entered password vs stored hash

5.2 Django CSRF Protection
    - All POST forms include {% csrf_token %} to prevent
      Cross-Site Request Forgery attacks

5.3 Session-Based Authentication
    - User login state stored in Django sessions (request.session)
    - Separate sessions for users ('username') and admins ('admin_username')
    - Session cleared on logout

5.4 Environment Variables
    - API keys and database credentials stored in .env file
    - Never hardcoded in source code
    - .env file should be added to .gitignore

6. HOW TO RUN THE PROJECT
=====================================================================

PREREQUISITES:
  - Python 3.10 or higher
  - MySQL Server installed and running
  - pip (Python package manager)

STEP 1: Navigate to project folder
    cd "Travel Plan Itinerary Generator/Travel Plan Itinerary Generator"

STEP 2: Install all dependencies
    pip install django pymysql python-dotenv groq google-generativeai torch transformers nltk

STEP 3: Create MySQL database
    Open MySQL Workbench and run:
    CREATE DATABASE itinerary;
    (Tables are auto-created when the server starts)

STEP 4: Create .env file in project root
    GROQ_API_KEY=your_groq_api_key
    GOOGLE_API_KEY=your_google_api_key
    MYSQL_HOST=localhost
    MYSQL_USER=root
    MYSQL_PASSWORD=your_mysql_password
    MYSQL_DATABASE=itinerary

STEP 5: Start the server
    python manage.py runserver 8080

STEP 6: Open in browser
    http://127.0.0.1:8080/

STOP THE SERVER:
    Press Ctrl + C in the terminal
